# Release Checklist

## Identity and store setup
- [ ] Confirm final game title
- [ ] Confirm package ID `com.oneapps.habibirun`
- [ ] Confirm developer identity is One Apps
- [ ] Replace the support-email placeholder everywhere
- [ ] Confirm public website, privacy, support, and press-kit URLs

## Art and listing
- [ ] Replace every concept asset with final approved artwork
- [ ] Capture genuine gameplay screenshots
- [ ] Confirm screenshots match the shipping build
- [ ] Native-review the Arabic localization
- [ ] Re-check live Play Console character limits
- [ ] Finalize category and contact details

## Android build
- [ ] Create and securely store the production signing key
- [ ] Export a signed Android App Bundle
- [ ] Verify current target Android API requirements
- [ ] Verify version code and version name
- [ ] Test on physical Android devices
- [ ] Inspect merged Android manifest permissions
- [ ] Inspect all SDKs, plugins, and dependencies
- [ ] Inspect release-build network traffic

## Policy and safety
- [ ] Verify privacy policy against the final binary
- [ ] Verify Google Play Data Safety answers
- [ ] Complete content rating after owner review
- [ ] Complete target audience after owner review
- [ ] Confirm app is not classified as child-directed without owner review
- [ ] Review app access declaration
- [ ] Review ads declaration
- [ ] Review government, financial, health, and news declarations if shown

> **Mandatory re-review:** The privacy policy and Google Play Data Safety form must be reviewed again if any of the following are introduced: ads, analytics, crash reporting, cloud saving, accounts, authentication, payments, leaderboards, social features, push notifications, external SDKs, device identifiers, or network logging.

## Testing and release
- [ ] Configure internal testing
- [ ] Upload AAB only after owner approval
- [ ] Review Play pre-launch report
- [ ] Fix crashes, ANRs, device compatibility issues, and layout problems
- [ ] Obtain owner approval before any production submission
- [ ] Submit production release only after explicit approval

## Content-rating preparation notes — do not submit automatically
- Fantasy combat
- Non-graphic enemy defeat
- Supernatural creatures
- Comedic tone
- No gambling
- No sexual content
- No realistic violence

## Play Console declarations to prepare but leave unsubmitted
- App access
- Ads
- Data Safety
- Content rating
- Target audience
- News app, if shown
- Government app, if shown
- Financial features, if shown
- Health app, if shown

## Testing-track plan
1. Internal test with trusted testers.
2. Resolve crashes and device issues.
3. Closed test if required by the developer account.
4. Review policy forms and listing one final time.
5. Prepare production only after explicit owner approval.
